There seems to be some difference, but with the smaller size of these groups (each with 21 data points), this certainly could be due to random variation.

We already have worked with tests like this using Welch’s two-sample t-test. However, since that test is parametric it assumes the distribution of sample means from samples of equal size to our data is normal. With larger sample sizes (as a rule of thumb n ≥ 30) this is taken care of automatically. However, given our smaller sample we can only assume this is the case if the population from which our data is drawn from is approximately normally distributed. To test if it’s plausible that this is the case we’ll run a Shapiro-Wilk test for normality and visualize our data using QQ-plots.

# Function to perform normality test and create QQ plot
perform_normality_test <- function(data, group_name, variable) {
  sample_data <- na.omit(data %>% filter(Treatment == group_name) %>% pull(variable))
  sample_size <- length(sample_data)
  
  if(sample_size < 50) {
    test_result <- shapiro.test(sample_data)
  } else {
    test_result <- ad.test(sample_data)
  }
  
  p2 <- ggplot(data.frame(sample_data), aes(sample = sample_data)) + 
    stat_qq() +
    stat_qq_line() +
    labs(title = paste("QQ Plot for", group_name), x = "Theoretical Quantiles", y = "Sample Quantiles") +
    theme_bw()
  
  list(plot = p2, test_result = test_result)
}

# Perform tests and generate QQ plots for each group
results_nonchilled <- perform_normality_test(Quebec_CO2, "nonchilled","uptake")
results_chilled <- perform_normality_test(Quebec_CO2, "chilled","uptake")

With p-values below .05 for both groups, we will reject the null hypothesis that our samples come from normally-distributed populations. So, it appears it may be unwise to use parametric tests like the two-sample t-test we’re used to. Though there is no cut-and-dry way to tell if the data is too skewed to mess up the results of our tests too much, it’s better to be safe than sorry and increase our chances of type I or II error.

This is where a Randomization or Permutation test comes to the rescue. Non-parametric tests like Randomization don’t need to assume the distribution of samples means is normal. They tend to rely instead on simulations to get the results we’re looking for.

Randomization works by first calculating the observed difference in the means between our two groups, in this case chilled and unchilled. It then takes the data and shuffles the categories, so that each data point is assigned to the chilled or unchilled group randomly, regardless of which group it belonged to in reality. Following this it takes the difference in group means between the chilled and unchilled groups, even though the data points assigned to these groups is purely random. It stores this value in the difference between means and repeats this process over many iterations (usually as common practice around 10,000 times). Our p-value is then found by calculating the number of iterations that had at least as extreme of a difference between means as our observed difference between groups contained. To re-iterate (as randomization is known for), the test is completed in the following steps:

1) Calculate the difference in means between both groups, in this case “chilled” and “unchilled”

2) Shuffle the categorical values (“chilled” or “unchilled”) randomly between the values of our data. In this case, each “uptake” value will be randomly assigned to the “chilled” or “unchilled” group.

3) Compute the difference in means of the two groups and add this value to a data frame.

4) Repeat steps 2 & 3 many times (in this case 10,000), until we have a set of 10,000 difference in means between our two groups + our actual observed difference

5) Calculate the proportion of shuffled groups that had, by pure random chance alone, a difference in group means at least as extreme as the difference we observed in reality.

We will then interpret this p-value the same as we always do. For a p-value > .05, we will fail to reject the null hypothesis. For a p-value < .05, we will reject, claiming there is sufficient evidence to suggest there is a significant difference between the two population group means.

Now, we will run our randomization simulation and graph the difference in means between all the simulated groups. Any values in red represent values that had a difference in group means at least as great as the one we observed.

# Function to perform the permutation test and generate a histogram
generate_permutation_plot <- function(data, n_permutations = 10000, binwidth = 0.5,p_size = 5) {
  all_uptake <- data$uptake
  n <- length(all_uptake)
  n_nonchilled <- sum(data$Treatment == "nonchilled")
  permuted_diffs <- numeric(n_permutations)
  
  observed_diff <- mean(data$uptake[data$Treatment == "chilled"]) - 
                   mean(data$uptake[data$Treatment == "nonchilled"])
  
  for (i in 1:n_permutations) {
    permuted <- sample(all_uptake)
    permuted_nonchilled <- permuted[1:n_nonchilled]
    permuted_chilled <- permuted[(n_nonchilled + 1):n]
    permuted_diffs[i] <- mean(permuted_chilled) - mean(permuted_nonchilled)
  }
  
  p_value <- mean(abs(permuted_diffs) >= abs(observed_diff))
  
  plot <- ggplot(data.frame(permuted_diffs), aes(x = permuted_diffs)) +
    geom_histogram(binwidth = binwidth, color = "black", 
                   aes(fill = abs(observed_diff) <= abs(permuted_diffs))) +
    labs(
      title = "Difference in Mean CO2 Uptake",
      x = "Difference in Mean CO2 Uptake",
      y = "Frequency"
    ) +
    theme_minimal() +
    theme(legend.position = "none") +
    annotate("text", x = Inf, y = Inf, label = paste("p-Value =", round(p_value, 4)), 
             hjust = 1.1, vjust = 2, size = p_size, color = "red", fontface = "bold")
  
  return(plot)
}

# Generate plots for four runs
generate_permutation_plot(Quebec_CO2)

We can see a large portion of our simulations yielded a difference in group means when the values in our data were assigned to a group completely randomly. Because of this, it doesn’t seem we have sufficient reason to claim that our observed difference is due to a true significant difference in the population means of each group. So, as we’re used to we will fail to reject the null hypothesis that there is no true difference between the chilled and unchilled treatment.

One thing to get used to that is common among simulation-based tests is that p-values are not fixed. We can re-run this randomization test multiple times and see that it will not be identical for each iteration. However, with a number of iterations great enough (10,000 in this case), the p-values will not stray far from each other. Though there is some slight difference, they converge to a common value.

# Generate plots for four runs
plot1 <- generate_permutation_plot(Quebec_CO2,p_size = 3.25)
plot2 <- generate_permutation_plot(Quebec_CO2,p_size = 3.25)

combined_plot <- ggarrange(plot1, plot2, plot3, plot4,

As stated, we can see that these simulations are unique, but the shape of the distributions remain similarly normal and the p-values are quite close to each other.

Perfect! We’ve run our randomization test, and despite the seeming non-normality of our population and small sample sizes, we can still find the information we desire for our test.

Non-parametric tests is a massive up-and-coming branch of statistics that may be the future of the discipline as a whole. Hopefully you can see the value of these intuitive and robust tests.
